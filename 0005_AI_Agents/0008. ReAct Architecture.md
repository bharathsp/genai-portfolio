# 🤖 ReAct (Reason + Act) Agent Architecture

ReAct is one of the **most important Agent architectures** and is the foundation behind many modern AI agents in:

* 🦜 LangChain
* 🕸️ LangGraph
* 🤖 AI Assistants
* 🔍 Research Agents
* 🛠️ Tool-Using Agents

---

# 🎯 What is ReAct?

**ReAct = Reason + Act**

The agent:

1. 🧠 **Reasons** about the problem
2. 🛠️ **Acts** by calling a tool
3. 👀 **Observes** the tool result
4. 🧠 **Reasons again**
5. 🔁 Repeats until it has the answer

---

## Simple Definition

> ReAct is an agent architecture where the LLM alternates between thinking (reasoning) and taking actions (tool calls) until it solves the task.

```text

```

---

```text

```

---




Final response:

```text
The weather is 32°C and sunny.
```

# ReAct in LangChain

Tools

```python
from langchain.tools import tool
```

---

Tool

```python
@tool
def calculator(a: int, b: int):
    return a * b
```

---

Agent

```python
agent = create_react_agent(
    model,
    tools=[calculator]
)
```

---

User

```python
agent.invoke(
    {
        "messages": [
            ("human", "What is 23 * 45?")
        ]
    }
)
```

---

Internal ReAct Cycle

```text
Thought:
Need calculator

Action:
calculator

Observation:
1035

Final Answer:
1035
```

# Interview Answer 🎤

> ReAct (Reason + Act) is an agent architecture where an LLM alternates between reasoning and tool usage. The agent first reasons about the problem, decides whether a tool is needed, executes the tool, observes the result, and then reasons again using the new information. This Thought → Action → Observation cycle continues until a final answer is produced. ReAct enables AI agents to perform multi-step reasoning, interact with external systems, and solve tasks more accurately than standalone LLMs.
