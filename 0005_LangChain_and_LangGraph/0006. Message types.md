# 💬 Messages in LangChain

In LangChain, conversations are represented using **Message Objects**.

Instead of sending plain strings, LangChain structures conversations as messages.

---

# 🏗️ Why Messages?

A real conversation contains different participants:

```text id="rlb9gv"
👨 User
🤖 AI Assistant
📋 Instructions
🛠️ Tools
```

LangChain models this using message types.

---

# The 4 Most Important Message Types

```text id="ylqyb7"
1️⃣ SystemMessage
2️⃣ HumanMessage
3️⃣ AIMessage
4️⃣ ToolMessage
```

# 1️⃣ 📋 SystemMessage

## What is it?

Defines the behavior, personality, and rules of the AI.

Think of it as:

> 📋 Instructions given to the AI before the conversation starts.

---

## Example

```python
from langchain_core.messages import SystemMessage

msg = SystemMessage(
    content="You are an expert Python teacher."
)
```

---

## Real-Life Example

Imagine hiring an employee.

Before they start work:

```text
You are a:
👨‍🏫 Teacher
Be patient
Give examples
```

That's the System Message.

---

## Example Conversation

### System

```text
You are a travel expert.
```

### User

```text
Suggest places in Goa.
```

The AI responds like a travel expert.

---

# 2️⃣ 👤 HumanMessage

## What is it?

Represents the user's input.

---

## Example

```python
from langchain_core.messages import HumanMessage

msg = HumanMessage(
    content="What is LangGraph?"
)
```

---

## Real-Life Example

Customer asks:

```text
What is Agentic AI?
```

This becomes:

```python
HumanMessage(
    content="What is Agentic AI?"
)
```

---

# 3️⃣ 🤖 AIMessage

## What is it?

Represents the model's response.

---

## Example

```python
from langchain_core.messages import AIMessage

msg = AIMessage(
    content="LangGraph is a framework..."
)
```

---

## Usually Created Automatically

```python
response = model.invoke(messages)
```

The response is typically:

```python
AIMessage(...)
```

---

## Example

User:

```text
What is LangChain?
```

AI:

```text
LangChain is a framework...
```

That AI response becomes an AIMessage.

---

# 4️⃣ 🛠️ ToolMessage

## What is it?

Represents the output returned from a tool.

Very important for Agentic AI.

---

## Example

Suppose the agent uses a calculator.

Tool:

```python
multiply(25, 40)
```

Output:

```text
1000
```

This output becomes a ToolMessage.

---

## Example

```python
from langchain_core.messages import ToolMessage

msg = ToolMessage(
    content="1000",
    tool_call_id="call_123"
)
```

# Full Conversation Example

```python
from langchain_core.messages import (
    SystemMessage,
    HumanMessage
)

messages = [
    SystemMessage(
        content="You are a helpful assistant."
    ),

    HumanMessage(
        content="Explain LangChain."
    )
]

response = model.invoke(
    messages
)

print(response.content)
```

# Code Example with Tool Calling

```python

    HumanMessage

    [
```


# Message History Example

Chatbots store messages as history.

```python
messages = [

    SystemMessage(
        content="You are a coding tutor."
    ),

    HumanMessage(
        content="What is Python?"
    ),

    AIMessage(
        content="Python is..."
    ),

    HumanMessage(
        content="Give example."
    )
]
```

The model sees the entire conversation.

# Interview Question 🎤

### Q: Why do we need different message types?

Because each message has a different purpose:

* 📋 **SystemMessage** → controls behavior
* 👤 **HumanMessage** → captures user input
* 🤖 **AIMessage** → stores model responses
* 🛠️ **ToolMessage** → stores tool results

This structured format allows LangChain agents to reason, use tools, and maintain conversation history effectively.

### One-Line Summary 🚀

> **Messages are the fundamental communication units in LangChain. `SystemMessage` sets behavior, `HumanMessage` represents user input, `AIMessage` contains model responses, and `ToolMessage` stores outputs returned by tools during agent execution.** 💬🦾
