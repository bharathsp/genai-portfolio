# 🕸️ Components of LangGraph

LangGraph is built around a few core building blocks:

```text
📦 State
🔵 Nodes
➡️ Edges
🕸️ Graph
```

Think of LangGraph as a **flowchart for AI agents**, where data moves through different steps until the task is completed.

---

# 🧠 1. State

## What is State?

**State is the shared memory of the graph.**

It contains all the information that nodes need to read or update while the workflow is running.

---

## Simple Definition

> State = Data that travels through the graph.

---

## Real-Life Analogy 📋

Imagine a hospital patient file.

```text
👤 Patient
    ↓
📋 Patient File
```

Doctors, nurses, and lab technicians all update the same file.

Similarly:

```text
🔵 Node 1
    ↓
📦 State
    ↓
🔵 Node 2
    ↓
📦 State
```

All nodes share and update the same state.

---

## Example

### State Definition

```python
from typing_extensions import TypedDict

class State(TypedDict):
    question: str
    answer: str
```

---

### State Value

```python
{
    "question": "What is LangGraph?",
    "answer": ""
}
```

---

## State Evolution

Initial:

```python
{
    "question": "What is LangGraph?",
    "answer": ""
}
```

After node execution:

```python
{
    "question": "What is LangGraph?",
    "answer": "LangGraph is a framework..."
}
```

---

# 🔵 2. Nodes

## What is a Node?

A node is a **unit of work**.

It performs some action and updates the state.

---

## Simple Definition

> Node = A step in the workflow.

---

## Real-Life Analogy 🍳

Making tea:

```text
Boil Water
Add Tea
Add Milk
Serve
```

Each step is a node.

---

## Example

```python
def generate_answer(state):
    return {
        "answer": "LangGraph is used for AI workflows."
    }
```

---

## Visualization

```text
📦 State
    ↓
🔵 Generate Answer Node
    ↓
📦 Updated State
```

---

## AI Agent Example

### Node 1

```text
Understand Question
```

### Node 2

```text
Search Web
```

### Node 3

```text
Generate Response
```

---

Flow:

```text
🔵 Understand
      ↓
🔵 Search
      ↓
🔵 Respond
```

---

# ➡️ 3. Edges

## What is an Edge?

Edges define how the graph moves from one node to another.

---

## Simple Definition

> Edge = Connection between nodes.

---

## Real-Life Analogy 🚦

Roads connect cities.

```text
🏙️ Chennai
      ↓
🛣️ Highway
      ↓
🏙️ Bangalore
```

Similarly:

```text
🔵 Node A
      ↓
➡️ Edge
      ↓
🔵 Node B
```

---

## Example

```python
graph.add_edge(
    "generate_answer",
    "final_response"
)
```

Meaning:

```text
generate_answer
       ↓
final_response
```

---

# Types of Edges

---

## Normal Edge

Always move forward.

```text
Node A
   ↓
Node B
```

---

## Conditional Edge

Choose path dynamically.

```text
         Success
             ↓
Node A → Node B

         Failure
             ↓
          Node C
```

---

## Example

```python
def route(state):
    if state["needs_search"]:
        return "search"
    return "answer"
```

---

Visualization:

```text
Question
    ↓

Need Search?
  /      \
Yes      No
 ↓        ↓

Search   Answer
```

# 🕸️ 4. Graph

## What is a Graph?

A graph is the complete workflow made up of:

* State
* Nodes
* Edges

---

## Simple Definition

> Graph = Entire AI workflow.

---

## Real-Life Analogy 🏭

Factory:

```text
Raw Material
      ↓
Assembly
      ↓
Testing
      ↓
Packaging
      ↓
Shipping
```

Complete factory workflow = Graph

---

## LangGraph Visualization

```text
          📦 State

              ↓

        🔵 Understand

              ↓

         🔵 Search

              ↓

        🔵 Summarize

              ↓

        🔵 Respond
```

---

# Building a Graph

```python
from langgraph.graph import StateGraph
```

---

Create Graph

```python
graph = StateGraph(State)
```

---

Add Nodes

```python
graph.add_node(
    "answer",
    generate_answer
)
```

---

Add Edge

```python
graph.add_edge(
    "answer",
    "__end__"
)
```

---

Compile

```python
app = graph.compile()
```

# Interview Answer 🎤

> LangGraph consists of four core components: **State, Nodes, Edges, and Graphs**. State represents the shared memory and data flowing through the workflow. Nodes are individual execution units that perform tasks and update the state. Edges define the transitions between nodes and can be static or conditional. The Graph combines states, nodes, and edges to create a complete AI workflow or agent execution pipeline.
