# 🚀 `graph.astream()` in LangGraph

After learning:

```python
graph.invoke()
graph.stream()
```

the next important method is:

```python
graph.astream()
```

This is the **asynchronous version of `graph.stream()`**.

---

# 🎯 Simple Definition

> **`graph.astream()` executes a LangGraph asynchronously and streams events in real time.**

Think:

```text
stream()
=
Synchronous Streaming

astream()
=
Asynchronous Streaming
```

---

# Understanding Sync vs Async

## Synchronous (stream)

```python
for event in graph.stream(...):
    print(event)
```

Flow:

```text
Task A
 ↓
Wait
 ↓
Task B
 ↓
Wait
 ↓
Task C
```

One operation at a time.

---

## Asynchronous (astream)

```python
async for event in graph.astream(...):
    print(event)
```

Flow:

```text
Task A ──┐
         │
Task B ──┼── Running Together
         │
Task C ──┘
```

Non-blocking execution.

---

# Why Use Async?

Because most Agentic AI systems perform:

```text
🌐 API Calls

🧠 LLM Calls

📦 Database Calls

🔍 Web Searches
```

These operations spend most of their time waiting.

---

Instead of:

```text
Call LLM
Wait

Call DB
Wait

Call API
Wait
```

Async allows:

```text
Call LLM
Call DB
Call API

Wait Once
```

Huge performance improvement.

# Stream Modes Work Here Too

Everything supported by:

```python
graph.stream()
```

```python

```

---

# Updates Mode

```python
async for event in graph.astream(
    inputs,
    stream_mode="updates"
):
    print(event)
```

Output:

```python
{
    "search":{
        "results":"..."
    }
}
```

---

# Values Mode

```python
async for event in graph.astream(
    inputs,
    stream_mode="values"
):
    print(event)
```

Output:

```python
{
    "question":"AI",
    "answer":"..."
}
```

Entire state after each node.

---

# Messages Mode

Most important for chat applications.

```python
async for chunk in graph.astream(
    inputs,
    stream_mode="messages"
):
    print(chunk)
```

Output:

```text
H
He
Hel
Hell
Hello
```

Token streaming.

---

# Interview Answer 🎤

> `graph.astream()` is the asynchronous streaming execution method in LangGraph. It behaves like `graph.stream()` but is designed for async environments such as FastAPI, WebSockets, and production AI applications. It emits intermediate graph events, node updates, state changes, and token streams in real time while allowing non-blocking execution and better scalability.
