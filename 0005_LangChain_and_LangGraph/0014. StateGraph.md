# 🕸️ StateGraph in LangGraph

## 1. What is StateGraph?

**StateGraph** is the most important class in LangGraph.

It is used to create workflows where:

* 📦 State (shared data) flows through the graph
* 🔵 Nodes read and update the state
* ➡️ Edges determine the execution path
* 🕸️ The graph orchestrates the entire workflow

---

## Simple Definition

> **StateGraph is a graph whose execution is driven by a shared state object.**

Think of it as:

```text
📦 State
    ↓
🔵 Node
    ↓
📦 Updated State
    ↓
🔵 Node
    ↓
📦 Updated State
```

---

# Why is it called StateGraph?

Because the entire workflow revolves around a shared **State**.

Unlike traditional workflows:

```text
Node A
   ↓
Node B
```

LangGraph workflows are:

```text
📦 State
   ↓
Node A
   ↓
📦 State
   ↓
Node B
   ↓
📦 State
```

The state is continuously updated.

# Complete Example

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph
from langgraph.graph import START, END

class State(TypedDict):
    question: str
    answer: str


def answer_node(state):
    return {
        "answer": f"You asked: {state['question']}"
    }

graph = StateGraph(State)

graph.add_node(
    "answer",
    answer_node
)

graph.add_edge(
    START,
    "answer"
)

graph.add_edge(
    "answer",
    END
)

app = graph.compile()
```

---


# Interview Answer 🎤

> **StateGraph is the core graph type in LangGraph. It manages workflows using a shared state object that flows between nodes. Each node reads and updates the state, while edges define the execution path. StateGraph supports branching, loops, conditional routing, memory management, and multi-step agent workflows, making it ideal for building Agentic AI systems.**
