# 🌊 `graph.stream()` in LangGraph

## 1. What is `graph.stream()`?

Normally when you execute a graph using:

```python
graph.invoke(...)
```

you only get the **final result**.

---

With:

```python
graph.stream(...)
```

you can see the graph executing **step by step in real-time**.

Think of it as:

```text
invoke()
=
Give me final answer

stream()
=
Show me how the answer is being produced
```

---

# Why Use Streaming?

Streaming is useful for:

### 👀 Debugging

See which node is running.

### 📊 Monitoring

Track graph execution.

### 🤖 Agent Observability

Understand agent reasoning.

### ⚡ Real-time Updates

Show progress to users.

# Basic Example

## State

```python
from typing_extensions import TypedDict

class State(TypedDict):
    question: str
    answer: str
```

---

## Node

```python
def answer_node(state):
    return {
        "answer": f"Answering {state['question']}"
    }
```

---

## Graph

```python
graph = StateGraph(State)

graph.add_node("answer", answer_node)

graph.add_edge(START, "answer")
graph.add_edge("answer", END)

app = graph.compile()
```

---

# invoke()

```python
result = app.invoke(
    {
        "question": "What is AI?"
    }
)

print(result)
```

Output:

```python
{
    "question":"What is AI?",
    "answer":"Answering What is AI?"
}
```

Only final result.

---

# stream()

```python
for event in app.stream(
    {
        "question":"What is AI?"
    }
):
    print(event)
```

Output:

```python
{
   "answer": {
      "answer":"Answering What is AI?"
   }
}
```

Node-by-node updates appear.

# What is Being Streamed?

By default:

```python
stream_mode="updates"
```

Streams node updates.

---

Flow:

```text
Node Executes
      ↓
State Update
      ↓
Event Emitted
```

---

# Stream Modes

LangGraph supports different stream modes.

---

# 1️⃣ updates

Most commonly used.

Streams node outputs.

```python
for event in graph.stream(
    inputs,
    stream_mode="updates"
):
    print(event)
```

Output:

```python
{
  "search":{
     "results":"..."
  }
}
```

---

# 2️⃣ values

Streams entire state after every node.

```python
for event in graph.stream(
    inputs,
    stream_mode="values"
):
    print(event)
```

---

Output

After Node 1:

```python
{
    "question":"What is AI?",
    "intent":"education"
}
```

---

After Node 2:

```python
{
    "question":"What is AI?",
    "intent":"education",
    "facts":"AI facts"
}
```

---

After Node 3:

```python
{
    "question":"What is AI?",
    "intent":"education",
    "facts":"AI facts",
    "answer":"..."
}
```

---

# Difference

## updates

Only changes.

```python
{
   "search":{
      "facts":"AI facts"
   }
}
```

---

## values

Entire state.

```python
{
   question,
   intent,
   facts,
   answer
}
```

---

# 3️⃣ messages

Useful for LLM token streaming.

---

Example:

```python
for chunk in graph.stream(
    inputs,
    stream_mode="messages"
):
    print(chunk)
```

---

Output:

```text
H
He
Hel
Hell
Hello
```

Token-by-token streaming.

# Multiple Stream Modes

You can combine modes.

```python
for event in graph.stream(
    inputs,
    stream_mode=["updates", "messages"]
):
    print(event)
```

---

This can show:

```text
Node Updates
+
LLM Tokens
```

simultaneously.

# invoke() vs stream()

| Feature              | invoke()   | stream() |
| -------------------- | ---------- | -------- |
| Final Result         | ✅          | ✅        |
| Intermediate Updates | ❌          | ✅        |
| Real-Time Monitoring | ❌          | ✅        |
| LLM Token Streaming  | ❌          | ✅        |
| Debugging            | ⚠️ Limited | ✅        |
| Agent Observability  | ⚠️ Limited | ✅        |

---

# Interview Answer 🎤

> `graph.stream()` executes a LangGraph workflow while emitting intermediate events in real time. Unlike `graph.invoke()`, which returns only the final state, `graph.stream()` allows developers to observe node execution, state updates, and model outputs as they happen. Common stream modes include `updates` (node changes), `values` (full state snapshots), and `messages` (token-level LLM streaming).
