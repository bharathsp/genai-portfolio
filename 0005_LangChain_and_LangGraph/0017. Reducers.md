# 🔄 Reducers in LangGraph

## 1. What is a Reducer?

A **Reducer** defines **how LangGraph should combine (merge) new state updates with the existing state**.

Whenever a node returns data, LangGraph needs to decide:

```text
Current State + New Update = ?
```

A reducer provides the answer.

---

## 🎯 Simple Definition

> A Reducer is a function that determines how state changes are applied.

Think:

```text
Current Value
      +
New Value
      ↓
Reducer
      ↓
Final Value
```

---

# 🏠 Real-Life Analogy

Imagine your bank account.

Current balance:

```text
₹1000
```

Salary credited:

```text
₹5000
```

Reducer logic:

```text
1000 + 5000 = 6000
```

The reducer determines how values are combined.

---

# Why Reducers are Needed?

Suppose state is:

```python
{
    "messages": ["Hello"]
}
```

A node returns:

```python
{
    "messages": ["Hi"]
}
```

Question:

```text
Should "Hi" replace "Hello"?
OR
Should it be added?
```

Reducers decide.

---

# Default Reducer (Replace)

If no reducer is specified:

```python
class State(TypedDict):

```

State:

```python

```

Node returns:

```python
{
    "answer": "New Answer"
}
```

Final state:

```python
{
    "answer": "New Answer"
}
```

The old value is replaced.

# 2️⃣ add_messages Reducer

Most popular reducer in LangGraph.

Import:

```python

```

class State(TypedDict):

# 8️⃣ Custom Reducer

You can build your own reducer.

---

Example

```python
def longest_answer(old, new):
    if len(new) > len(old):
        return new
    return old
```

---

Result

```text
Keep the longest answer
```

# Using Reducers with Annotated

Reducers are usually attached using:

    total_tokens: Annotated[

# Interview Answer 🎤

> Reducers in LangGraph define how state updates from nodes are merged into the existing state. By default, values are replaced, but reducers enable custom merge behavior such as appending messages (`add_messages`), accumulating lists, summing numbers, merging dictionaries, removing duplicates, or implementing custom business logic. Reducers are attached to state fields using Python's `Annotated` type and are essential for maintaining memory, aggregation, and collaborative workflows in agentic systems.
