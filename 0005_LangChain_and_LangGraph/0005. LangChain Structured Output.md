# 📦 Structured Output in LangChain

## What is Structured Output?

Normally, an LLM returns plain text:

```text
The customer's name is John.
Age is 25.
City is Chennai.
```

This is difficult for applications to process.

---

With **Structured Output**, the LLM returns data in a predefined structure:

```python
{
    "name": "John",
    "age": 25,
    "city": "Chennai"
}
```

This makes it easy to:

* 📊 Store in databases
* 🌐 Send via APIs
* 🧠 Feed into other agents
* 📈 Process programmatically

---

# Why Structured Output?

Without structured output:

```text
🤖 AI Response

Name: John
Age: Twenty Five
Lives in Chennai
```

You need string parsing.

---

With structured output:

```python
{
    "name": "John",
    "age": 25,
    "city": "Chennai"
}
```

Directly usable.

---

# Three Common Ways

```text

```

# 1️⃣ Pydantic (Most Popular)

## What is Pydantic?

Pydantic is a Python library for:

* ✅ Data validation
* ✅ Type checking
* ✅ Structured data
* ✅ JSON serialization

---

## Most Important Feature

### Runtime Validation ✅

Pydantic validates data when your application runs.

---

Example:

```python
from pydantic import BaseModel

class Person(BaseModel):
    name: str
    age: int
```

---

Valid:

```python
person = Person(
    name="John",
    age=25
)
```

---

Invalid:

```python
person = Person(
    name="John",
    age="abc"
)
```

Output:

```text
ValidationError
```

Pydantic catches the issue immediately.

---

# BaseModel

All Pydantic models inherit from:

```python
from pydantic import BaseModel
```

Example:

```python
from pydantic import BaseModel

class Employee(BaseModel):
    id: int
    name: str
```

---

# Field()

Used for:

* Description
* Validation
* Default values

---

Example:

```python
from pydantic import BaseModel, Field

class Employee(BaseModel):
    id: int = Field(
        description="Employee ID"
    )

    name: str = Field(
        description="Employee Name"
    )
```

---

# Real-Life Example

Customer extraction:

```python
from pydantic import BaseModel, Field

class Customer(BaseModel):
    name: str = Field(
        description="Customer name"
    )

    email: str = Field(
        description="Customer email"
    )

    age: int = Field(
        description="Customer age"
    )
```

---

# Nested Structures

Pydantic supports nested models.

---

Example

## Address

```python
from pydantic import BaseModel

class Address(BaseModel):
    city: str
    state: str
```

---

## Customer

```python
class Customer(BaseModel):
    name: str
    address: Address
```

---

Usage:

```python
customer = Customer(
    name="John",
    address={
        "city":"Chennai",
        "state":"Tamil Nadu"
    }
)
```

Output:

```python
{
    "name":"John",
    "address":{
        "city":"Chennai",
        "state":"Tamil Nadu"
    }
}
```

---

# LangChain Example

```python
from pydantic import BaseModel

class Person(BaseModel):
    name: str
    age: int

structured_llm = model.with_structured_output(
    Person
)

response = structured_llm.invoke(
    "John is 25 years old"
)

print(response)
```

Output:

```python
Person(
    name="John",
    age=25
)
```

# Why Pydantic is Preferred?

✅ Runtime Validation

✅ Strong Type Safety

✅ Nested Objects

✅ JSON Support

✅ Production Ready

# 2️⃣ TypedDict

## What is TypedDict?

A type-hinting mechanism from Python.

Provides structure hints but:

❌ No runtime validation

---

Example

```python
from typing import TypedDict

class Person(TypedDict):
    name: str
    age: int
```

---

Usage

```python
person = {
    "name":"John",
    "age":25
}
```

---

Looks good.

---

But this also works:

```python
person = {
    "name":"John",
    "age":"abc"
}
```

No error!

---

Why?

Because TypedDict only helps:

```text
IDE
Type Checker
Static Analysis
```

It does NOT validate during execution.

---

# Real-Life Analogy

TypedDict:

```text
📋 Checklist

Age should be integer
```

But nobody checks it.

---

Pydantic:

```text
📋 Checklist
+
👮 Inspector
```

Invalid values are rejected.

---

# LangChain Example

```python
from typing import TypedDict

class Person(TypedDict):
    name: str
    age: int

structured_llm = model.with_structured_output(
    Person
)
```

Works well for lightweight schemas.

---

# Advantages of TypedDict

✅ Simple

✅ Lightweight

✅ No dependency

✅ Fast

---

# Limitations

❌ No runtime validation

❌ No advanced constraints

❌ Weak error handling

---

# 3️⃣ Dataclass

## What is Dataclass?

Python's built-in way of creating data containers.

---

Example

```python
from dataclasses import dataclass

@dataclass
class Person:
    name: str
    age: int
```

---

Usage

```python
person = Person(
    "John",
    25
)
```

---

Output

```python
Person(
    name='John',
    age=25
)
```

---

# Does Dataclass Validate?

No.

Example:

```python
person = Person(
    "John",
    "abc"
)
```

No error.

---

Because:

```text
Dataclass
=
Convenient Container

NOT Validator
```

---

# Real-Life Analogy

Dataclass:

```text
📦 Storage Box

Stores anything.
```

Pydantic:

```text
📦 Smart Storage Box

Checks contents before storing.
```

---

# Nested Dataclass Example

```python
from dataclasses import dataclass

@dataclass
class Address:
    city: str

@dataclass
class Person:
    name: str
    address: Address
```

# Interview Answer 🎤

> **Structured output ensures LLM responses follow a predefined schema instead of returning free-form text. In LangChain, the most common schema types are Pydantic, TypedDict, and Dataclass. Pydantic is preferred because it provides runtime validation, supports nested structures, and catches invalid data. TypedDict provides type hints only and has no runtime validation, while Dataclass is a lightweight Python container that stores structured data but does not validate it.**

---

# 🎯 One-Line Summary

```text
Pydantic = Structured Data + Runtime Validation ✅

TypedDict = Structured Data + Type Hints Only ⚠️

Dataclass = Structured Data Container 📦
```

For most real-world LangChain and Agentic AI applications, **Pydantic is the preferred choice** because of its validation and reliability. 🚀🧠📦
