# 🤖 `model.profile` in LangChain

## What is `model.profile`?

`model.profile` provides **metadata (information) about the model** that LangChain knows about.

Think of it as:

> 📋 **Model Profile = Identity Card of the LLM**

It helps you understand:

* 🤖 Model name
* 🏢 Provider
* 🖼️ Image support
* 📝 Structured output support
* 🛠️ Tool calling support
* 📏 Context window
* 🎥 Multimodal capabilities
* Other model features

---

# Simple Example

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(
    model="gpt-4o-mini"
)

print(model.profile)
```

Output (varies by model/version):

```python
{
    "model_name": "gpt-4o-mini",
    "tool_calling": True,
    "structured_output": True,
    "image_input": True,
    "audio_input": False,
    ...
}
```

---

# Why is it Useful?

Suppose you're building a generic AI application.

Different models have different capabilities.

| Model             | Tool Calling | Image Input |
| ----------------- | ------------ | ----------- |
| GPT-4o            | ✅            | ✅           |
| GPT-4o-mini       | ✅            | ✅           |
| Some older models | ❌            | ❌           |

Instead of hardcoding assumptions, you can inspect the model profile.

---

# Example: Check Tool Support

```python
if model.profile.get("tool_calling"):
    print("Tool calling supported")
else:
    print("Tool calling not supported")
```

---

# Example: Check Vision Support

```python
if model.profile.get("image_input"):
    print("Can process images")
```

---

# Real-World Scenario 1: Agent Builder 🦾

Suppose you're building a framework that allows users to choose any model.

Before creating an agent:

```python
if model.profile.get("tool_calling"):
    agent = model.bind_tools(tools)
else:
    raise Exception(
        "Selected model does not support tools"
    )
```

---

# Real-World Scenario 2: Image Analysis App 🖼️

User uploads an image.

Your application checks:

```python
if model.profile.get("image_input"):
    analyze_image()
else:
    switch_to_vision_model()
```

---

# Real-World Scenario 3: Structured Output 📄

Suppose you want JSON output.

```python
if model.profile.get("structured_output"):
    use_json_mode()
```

# Common Attributes You May See

Depending on provider and LangChain version:

| Property             | Meaning              |
| -------------------- | -------------------- |
| 🤖 model_name        | Name of model        |
| 🏢 provider          | OpenAI, Google, Groq |
| 🛠️ tool_calling     | Supports tools       |
| 📝 structured_output | Supports JSON output |
| 🖼️ image_input      | Supports images      |
| 🔊 audio_input       | Supports audio       |
| 🎥 video_input       | Supports video       |
| 📏 context_window    | Maximum tokens       |
| 🌊 streaming         | Supports streaming   |

# Practical Use Case: Dynamic Capability Detection

```python
profile = model.profile

      profile.get("tool_calling"))

      profile.get("image_input"))

      profile.get("structured_output"))
```

Output:

```text

Image Input: True
Structured Output: True
```

# Model Profile vs Model Response

### `model.profile`

```python
print(model.profile)
```

Returns:

```text
Model capabilities
```

---

### `model.invoke()`

```python
model.invoke(
    "Explain LangChain"
)
```

Returns:

```text
Actual AI response
```

## One-Line Summary 🚀

> **`model.profile` is LangChain's way of exposing a model's capabilities and metadata (tool calling, image support, structured output, streaming, etc.), allowing applications to dynamically decide how to use a model.** 🧠📋🤖
