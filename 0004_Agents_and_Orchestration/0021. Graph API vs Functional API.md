# 🕸️ LangGraph Workflows: Graph API vs Functional API

LangGraph provides **two main ways** to build workflows:

```text
1️⃣ Graph API
2️⃣ Functional API
```

Both can build AI agents and workflows, but they differ in how you define the execution flow.

---

# 🎯 Simple Overview

| Feature             | Graph API               | Functional API   |
| ------------------- | ----------------------- | ---------------- |
| Style               | Explicit Graph          | Python Functions |
| Nodes & Edges       | Manually defined        | Implicit         |
| Best For            | Complex Agent Workflows | Simple Workflows |
| Conditional Routing | Excellent               | Good             |
| Multi-Agent Systems | Excellent               | Possible         |
| Visualization       | Excellent               | Limited          |
| Learning Curve      | Higher                  | Lower            |

---

# 🕸️ 1. Graph API

## What is Graph API?

Graph API is the **core LangGraph approach** where you explicitly define:

* 📦 State
* 🔵 Nodes
* ➡️ Edges
* 🔀 Conditional Edges

You build a graph structure yourself.

---

## Real-Life Analogy 🏙️

Imagine designing a metro map.

```text
Station A
    ↓
Station B
    ↓
Station C
```

You explicitly define:

* Stations = Nodes
* Tracks = Edges

---

## Basic Flow

```text
START
  ↓
Understand
  ↓
Search
  ↓
Answer
  ↓
END
```

---

## Example

### State

```python
from typing_extensions import TypedDict

class State(TypedDict):
    question: str
    answer: str
```

---

### Node

```python
def answer_node(state):
    return {
        "answer": f"Answering: {state['question']}"
    }
```

---

### Build Graph

```python
from langgraph.graph import (
    StateGraph,
    START,
    END
)

graph = StateGraph(State)

graph.add_node(
    "answer",
    answer_node
)

graph.add_edge(
    START,
    "answer"
)

graph.add_edge(
    "answer",
    END
)

app = graph.compile()
```

---

## Visualization

```text
START
  ↓
🔵 answer
  ↓
END
```

# Functional API

## What is Functional API?

Instead of building nodes and edges manually, you write workflows as normal Python functions.

LangGraph manages orchestration behind the scenes.

---

## Simple Definition

> Functional API lets you define workflows using Python functions and decorators instead of manually creating graphs.

---

# Real-Life Analogy 🍔

Graph API:

```text
Build Kitchen Layout
Build Counters
Build Cooking Stations
Build Delivery System
```

---

Functional API:

```text
Just write recipe steps
```

Much simpler.

---

# Basic Idea

Instead of:

```python
graph.add_node(...)
graph.add_edge(...)
```

you write:

```python
@entrypoint()
def workflow():
    ...
```

---

# Example

```python
from langgraph.func import entrypoint

@entrypoint()
def greet(name: str):

    return f"Hello {name}"
```

Usage:

```python
result = greet.invoke("John")
```

Output:

```python
Hello John
```

---

# Functional Workflow Example

```python
from langgraph.func import entrypoint

@entrypoint()
def customer_support(question):

    answer = f"Answering {question}"

    return answer
```

---

Visualization

```text
Question
   ↓
Process
   ↓
Answer
```

No graph definition required.

---

# Functional API with Multiple Steps

```python
from langgraph.func import entrypoint

@entrypoint()
def workflow(question):

    step1 = understand(question)

    step2 = search(step1)

    step3 = summarize(step2)

    return step3
```

---

Visualization

```text

    ↓
Search
    ↓
Summarize
```

---

## Functional API

# Interview Answer 🎤

> LangGraph provides two workflow creation approaches: Graph API and Functional API. The Graph API explicitly defines states, nodes, edges, and routing logic, making it ideal for complex agentic workflows, multi-agent systems, loops, and conditional execution. The Functional API allows developers to create workflows using standard Python functions and decorators, making it simpler and faster for straightforward workflows. Graph API offers greater flexibility and control, while Functional API prioritizes simplicity and developer productivity.
