# рҹӨ– Model Calling in LangChain

Once you've initialized a model using:

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(
    model="gpt-4o-mini"
)
```

There are **3 major ways** to call the model:

```text
1пёҸвғЈ invoke()    вҶ’ One request at a time
2пёҸвғЈ stream()    вҶ’ Token-by-token streaming
3пёҸвғЈ batch()     вҶ’ Multiple requests together
```

Think of them as:

| Method       | Real Life Analogy                    |
| ------------ | ------------------------------------ |
| рҹ“һ invoke()  | One phone call                       |
| рҹҺҷпёҸ stream() | Live conversation                    |
| рҹ“Ұ batch()   | Process a stack of requests together |

---

# 1пёҸвғЈ `invoke()` вҖ” Single Request

## What is it?

The simplest and most commonly used method.

You send one input and receive one response.

---

## Syntax

```python
response = model.invoke(
    "Explain LangGraph"
)

print(response.content)
```

---

## Flow

```text
рҹ‘Ө User Question
       в”Ӯ
       в–ј
   invoke()
       в”Ӯ
       в–ј
   рҹ§  Model
       в”Ӯ
       в–ј
 Complete Response
```

---

## Real-Life Example

Asking ChatGPT:

> Explain Agentic AI

You wait.

Then receive the full answer.

---

## Example

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(
    model="gpt-4o-mini"
)

response = model.invoke(
    "What is LangChain?"
)

print(response.content)
```

---

## When to Use

вң… Chatbots

вң… Q&A systems

вң… Small applications

вң… Testing prompts

---

# 2пёҸвғЈ `stream()` вҖ” Streaming Response

## What is it?

Instead of waiting for the entire response, tokens arrive gradually.

Similar to ChatGPT typing.

---

## Syntax

```python
for chunk in model.stream(
    "Explain Agentic AI"
):
    print(chunk.content, end="")
```

---

## Flow

```text
рҹ‘Ө User
    в”Ӯ
    в–ј
 stream()
    в”Ӯ
    в–ј
рҹ§  Model
    в”Ӯ
    в–ј
"A..."
"Ag..."
"Agent..."
```

---

## Example

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(
    model="gpt-4o-mini"
)

for chunk in model.stream(
    "Explain LangGraph"
):
    print(chunk.content, end="")
```

---

## Output

Instead of:

```text
LangGraph is ...
```

You get:

```text
L
La
Lang
LangG
LangGr
...
```

(streamed internally as tokens)

---

## Real-Life Analogy рҹҺҷпёҸ

### invoke()

```text
рҹ“һ Phone Call

You ask question.

Wait.

Get complete answer.
```

---

### stream()

```text
рҹҺӨ Live Speech

Speaker starts talking immediately.

```

---

## Benefits

вң… Faster perceived response

вң… Better user experience

вң… Great for chat applications

вң… Ideal for long answers

---

## Used In

рҹ’¬ ChatGPT

рҹӨ– AI Assistants

# 3пёҸвғЈ `batch()` вҖ” Multiple Requests

## What is it?

Processes many requests together.

---

## Example

Instead of:

```python
model.invoke("What is AI?")
model.invoke("What is ML?")
model.invoke("What is DL?")
```

Use:

```python
responses = model.batch([
    "What is AI?",
    "What is ML?",
    "What is DL?"
])
```

---

## Flow

```text
Question 1
Question 2
Question 3
     в”Ӯ
     в–ј
   batch()
     в”Ӯ
     в–ј
   рҹ§  Model
     в”Ӯ
     в–ј
Response 1
Response 2
Response 3
```

---

## Example

```python
questions = [
    "What is AI?",
    "What is ML?",
    "What is Deep Learning?"
]

responses = model.batch(
    questions
)

for r in responses:
    print(r.content)
```

---

## Real-Life Analogy рҹ“Ұ

### invoke()

```text
Post 1 letter
Wait
```

---

### batch()

```text
Put 100 letters
into one courier pickup
```

Much more efficient.

# вҡЎ `max_concurrency`

One of the most important options with `batch()`.

---

## What is it?

Controls how many requests can run in parallel.

---

## Example

```python
responses = model.batch(
    questions,
    config={
        "max_concurrency": 5
    }
)
```

---

## Meaning

```text
max_concurrency = 5
```

LangChain can process:

```text
Request 1
Request 2
Request 3
Request 4
Request 5
```

at the same time.

---

## Visualization

### max_concurrency = 1

```text
Req1
 вҶ“
Req2
 вҶ“
Req3
 вҶ“
Req4
```

Sequential.

---

### max_concurrency = 4

```text
Req1 в”Җв”җ
Req2 в”Җв”Ө
Req3 в”Җв”Ө
Req4 в”Җв”ҳ
```

Parallel.

---

## Example

```python
questions = [
    "Explain AI",
    "Explain ML",
    "Explain NLP",
    "Explain RAG",
    "Explain Agents"
]

responses = model.batch(
    questions,
    config={
        "max_concurrency": 3
    }
)
```

---

## Real-Life Analogy рҹҸӯ

Imagine a factory.

### max_concurrency = 1

```text

```

Slow.

---

### max_concurrency = 10

```text

```

Faster.

# рҹ“Ҡ Performance Comparison

* рҹ“һ `invoke()` вҶ’ simplest, one request

* рҹ“Ұ `batch()` вҶ’ highest throughput for many requests

# рҹҺҜ When Should You Use Each?

## рҹ“һ invoke()

Use when:

* Single user query
* Chatbots
* Testing prompts

```python
model.invoke(...)
```

---

## рҹҺҷпёҸ stream()

Use when:

* Long responses
* Chat interfaces
* Better UX

```python
model.stream(...)
```

---

## рҹ“Ұ batch()

Use when:

* Processing many prompts
* Bulk summarization
* Bulk classification
* Evaluation datasets

```python
model.batch(...)
```

## Quick Interview Answer рҹҺӨ

> **`invoke()` is used for a single model call, `stream()` returns tokens progressively for real-time user experiences, and `batch()` processes multiple inputs together. `max_concurrency` controls how many batch requests can execute in parallel, improving throughput and performance.** рҹҡҖ
