# 🧩 Glue Code

**Glue Code** is a very common term in AI, MCP, LangChain, LangGraph, APIs, and software integration.

---

# 🎯 What is Glue Code?

## Simple Definition

> **Glue Code is the code that connects different systems, components, tools, APIs, or libraries together.**

It doesn't usually contain complex business logic.

Instead, it acts as a bridge.

---

# 🏠 Real-Life Analogy

Imagine building a house.

You have:

```text
🧱 Bricks
🚪 Doors
🪟 Windows
🏠 Roof
```

But these components won't form a house by themselves.

You need:

```text
🧱 Cement
```

to connect everything.

Similarly:

```text
LLM
Database
API
Tool
Vector DB
```

need

```text
🧩 Glue Code
```

to work together.

---

# Example Without Glue Code

You have:

```python
llm = ChatOpenAI()

db = Database()

email_service = EmailService()
```

All exist independently.

```text
LLM          Database          Email
 │               │              │
 ❌ No connection
```

---

# Example With Glue Code

```python
user_question = "Send revenue report"

report = db.get_report()

summary = llm.invoke(
    f"Summarize: {report}"
)

email_service.send(summary)
```

This code connects:

```text
Database
   ↓
LLM
   ↓
Email Service
```

The connecting code is the glue code.

---

# AI Agent Example

Suppose an agent needs:

```text
🌐 Web Search
📄 PDF Reader
🧠 LLM
📧 Email Tool
```

---

Glue code coordinates everything:

```python
search_results = search_tool.invoke(query)

pdf_content = pdf_reader.invoke(file)

response = llm.invoke(
    search_results + pdf_content
)

email_tool.invoke(response)
```

# MCP Example

Remember MCP architecture:

```text
LLM
 ↓
MCP Host
 ↓
MCP Client
 ↓
MCP Server
 ↓
Database
```

The MCP framework handles much of the integration, but developers still write glue code.

Example:

```python
server = FastMCP("CRM Server")

@server.tool()
def get_customer(id):
    return db.query(id)
```

This tool acts as glue between:

```text
MCP Server
      ↓
Database
```

# LangChain Example

Suppose we have:

```python

vector_db
```

Glue code:

```python
docs = retriever.invoke(question)

response = llm.invoke(
    f"""
    Context:
    {docs}

    Question:
    {question}
    """
)
```

---

Architecture:

```text
Question
   ↓
Retriever
   ↓
Documents
   ↓
LLM
   ↓
Answer
```

The connecting logic is glue code.

# LangGraph Example

```python
graph.add_edge(

    "search",
```

# Why Is It Called Glue Code?

Because it "glues" components together.

```text
Tool A
  │
  ▼

🧩 Glue Code

  ▲
  │

Tool B
```

Without glue code:

```text
Independent Systems
```

With glue code:

```text
Integrated System
```

---

# Characteristics of Glue Code

### ✅ Connects Components

```text
LLM ↔ Tool
```

---

### ✅ Handles Data Transformation

Example:

```python
json_data
```

to

```python
HumanMessage
```

---

### ✅ Orchestrates Workflow

```text
Search
 ↓
Analyze
 ↓
Summarize
```

---

### ✅ Usually Not Core Business Logic

Business logic:

```python
calculate_tax()
```

Glue code:

```python
tax = calculate_tax()

save_to_db(tax)

send_email(tax)
```

# Advantages

## 🚀 Reuse Existing Systems

No need to rewrite tools.

---

## 🔌 Easy Integrations

Connect multiple services.

---

## 🤖 Enables Agent Workflows

Most agent systems are mostly glue code.

---

## 🏗️ Faster Development

Combine existing components quickly.

---

# Drawback

Sometimes projects become:

```text
90% Glue Code
10% Business Logic
```

when integrating many systems.

This can make maintenance harder.

---

# Interview Answer 🎤

> Glue Code is the code that connects different software components, services, APIs, databases, tools, or libraries so they can work together as a complete system. It usually contains orchestration, data transformation, routing, and integration logic rather than core business logic. In AI systems, glue code often connects LLMs, tools, vector databases, APIs, MCP servers, and agent workflows.
