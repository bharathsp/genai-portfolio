## ⚙️ What are **PEFT Adapters**?

**Adapters** are small, trainable neural network modules 🧩 that you **insert into a frozen large language model (LLM)** to teach it new tasks — **without retraining the whole model**.

> 🧠 Think of them as **plug-ins for LLMs** — tiny layers that modify the model’s behavior efficiently, instead of rewriting its entire brain.

---

## 💡 Why Use Adapters?

| 🏷️ Challenge                                          | 💡 Adapter Solution                                       |
| ------------------------------------------------------ | --------------------------------------------------------- |
| Full fine-tuning updates **billions** of parameters 💸 | Adapters train only a **few million** parameters 🪶       |
| Hard to maintain multiple task versions                | Each adapter = a separate task “module” 🔌                |
| Expensive GPU & storage usage                          | Low compute and memory footprint ⚙️                       |
| Need to retain base model knowledge                    | Base model stays frozen 🧊; adapters add new capabilities |

---

## 🧩 How Adapters Work

When fine-tuning, instead of updating the whole LLM layer,
you **insert adapter layers** inside existing transformer blocks (e.g., after feed-forward or attention layers).

### Concept Diagram

```
Original Transformer Layer
┌──────────────────────────────┐
│ Input → Linear → Activation → Output │
└──────────────────────────────┘

↓ With Adapter Added ↓

┌──────────────────────────────┐
│ Input                        │
│  ↓                           │
│ Linear → Activation → Output │
│   ↓                          │
│  Adapter Layer 🧩 (trainable) │
└──────────────────────────────┘
```

✅ Base weights = **Frozen**
✅ Adapter weights = **Trainable**
✅ Result = Efficient fine-tuning 🎯

---

## 🧱 Structure of an Adapter

An adapter usually has two main layers:

```
Adapter(x) = x + W_up( ReLU( W_down(x) ) )
```

Where:

* **W_down** → reduces the dimension (compression step)
* **ReLU** → introduces non-linearity
* **W_up** → expands it back to original size

So, instead of training huge matrices, adapters train these **small bottleneck layers**.

---

## ⚡ Types of PEFT Adapters

| 🧩 Adapter Type      | 🔍 Description                                  | 💪 Use Case                          |
| -------------------- | ----------------------------------------------- | ------------------------------------ |
| **Standard Adapter** | Adds trainable modules to transformer layers    | General domain adaptation            |
| **LoRA**             | Uses low-rank matrices instead of full adapters | Memory-efficient fine-tuning         |
| **Prefix Tuning**    | Adds special “learnable prompts” to model input | Instruction or conversational tuning |
| **Prompt Tuning**    | Optimizes only a few virtual tokens             | Lightweight personalization          |
| **BitFit**           | Trains only bias terms of the model             | Ultra-lightweight tuning             |

All of these fall under the **PEFT (Parameter-Efficient Fine-Tuning)** umbrella.

## 🚀 Benefits of Using Adapters

| ✅ Benefit                  | 💬 Explanation                                     |
| -------------------------- | -------------------------------------------------- |
| 💾 **Lightweight**         | Only a few MB per task vs GBs for full fine-tuning |
| ⚡ **Fast Training**        | Fewer parameters → faster convergence              |
| 🔁 **Modular**             | Easily swap adapters for different domains/tasks   |
| 🔒 **Preserves Knowledge** | Base LLM remains unchanged                         |
| 🧠 **Scalable**            | Fine-tune multiple tasks on one model efficiently  |
